# 模块 0：🎤 语音输入处理

处理两种语音输入场景：
1. **语音命令**：用户用语音说出操作指令（如"帮我分析这篇论文"）
2. **音频文件转写**：用户上传了含论文朗读/摘要的音频文件，需先转为文字

---

## 场景 A：语音命令识别（无需 MCP）

Claude Desktop 内置语音转文字，用户说话后文字会自动出现在对话框。
**无需额外配置**，技能正常接收文字后处理即可。

### 语音命令映射表

识别以下口语表达并映射到对应模块，**跳过菜单直接执行**：

| 用户可能说的话（中英文） | 执行模块 |
|----------------------|---------|
| 分析 / analyze / 帮我看看 / deep dive / 读一下 | 模块 1：深度分析 |
| 思维导图 / mind map / 脑图 / visualize / 画个图 | 模块 2：思维导图 |
| 审稿 / review / 帮我审 / peer review / 提意见 | 模块 3：审稿意见 |
| 宣传 / promote / 推文 / tweet / 公众号 / 介绍一下 | 模块 4：宣传脚本 |
| 播客 / podcast / 做成播客 / 音频 / audio | 模块 5：播客音频 |
| 导出 / export / google doc / 保存 / save | 模块 6：导出 Google Docs |
| 全部 / all / 都做 / everything / 一起来 | 模块 1→5 全部 |

**模糊匹配原则**：语音识别可能产生错别字或不完整词语，理解用户意图优先，不强求精确匹配。

---

## 场景 B：音频文件转写（需 Whisper MCP）

用户上传了音频文件（MP3、M4A、WAV 等），内容是论文摘要或朗读内容。

### 第零步：检测 Whisper MCP 是否可用

尝试调用 `whisper_transcribe` 或 `stt_transcribe` 工具：

- ✅ **调用成功** → MCP 已连接，进入转写流程
- ❌ **调用失败** → 提示安装引导，并建议替代方案

---

### Whisper MCP 未连接时的提示

> ⚠️ **未检测到语音转写 MCP，无法直接处理音频文件。**
>
> **替代方案（选一即可）：**
>
> **方案 1：使用 Claude Desktop 内置语音输入**
> 点击对话框旁的 🎤 麦克风图标，直接朗读论文摘要，Claude 会自动转写。
>
> **方案 2：安装 Whisper MCP**
>
> 确认已安装 Python 和 pip，然后：
> ```bash
> pip install openai-whisper
> ```
>
> 在 Claude Desktop 配置文件中添加：
> ```json
> "whisper": {
>   "command": "python",
>   "args": ["-m", "whisper_mcp"]
> }
> ```
>
> **方案 3：用其他工具先转为文字**
> - macOS：「备忘录」→ 语音输入
> - Windows：Windows 语音识别
> - 在线工具：[whisper.ai](https://whisper.ai) 等
>
> 转为文字后直接粘贴到对话框即可。

---

### Whisper MCP 可用时的转写流程

1. 调用 `whisper_transcribe`，参数：
   - `file_path`：用户上传的音频文件路径
   - `language`：`zh`（中文）或 `en`（英文），不确定时留空自动检测

2. 转写完成后，将文字作为论文内容，**返回 SKILL.md 第二步**继续处理

3. 告知用户：
   > "🎤 音频转写完成，识别到以下内容：
   > [转写结果前100字]...
   > 正在分析论文内容，请稍候。"

4. 若转写结果不完整（识别字数 < 100），提示：
   > "转写内容较短，可能是音频质量问题。建议直接粘贴论文文本以获得更好效果。"

---

## 通用语音输入原则

- **容错优先**：语音识别结果可能有错字，优先理解语义，不因错字报错
- **双语兼容**：中英文混合语音命令均支持
- **无论哪种方式**，最终都将内容交回 SKILL.md 主流程处理
